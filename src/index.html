How Long will a Rookie Last in the NBA?

This lesson will teach you the K-Nearest Neighbors machine laerning algorithm.

  • We'll use a dataset about rookie (1st year) NBA players' career lengths.

  • I'll introduce you to one kind of prediction problem that computers solve, called a classification task.

  • We'll use the K-Nearest Neighbors (KNN) algorithm to predict whether or not a rookie basketballer will have a career in the NBA that lasts at least 5 years.

  • You'll tune our KNN model's hyperparametersThis is just a fancy word for a model's settings..

  • We'll measure how well our model does at predicting which players will have 5+ year long NBA careers.

The Dataset

For this lesson, we’ll use this datasetLink to dataset of property sales. It has one row per property sold, and columns with some information about each property.

Take a moment to explore the dataset.


Classification

A common task in machine learning is to train a computer to accurately guess whether a new mystery datapoint belongs in one group (also called a class) or another.

In our dataset, each row is either a house or an apartment. These 2 groups (house & apartment) will be our classes.

Mystery Values

This data comes from a popular real estate listings website. Sometimes, the website’s users don’t fill out the 'property type' field, so there are some rows that have a missing value in that column.

How could we train a computer to predict whether a new row is a house or an apartment?

Classifying Mystery Values

The task above is asking us to train a computer to determine the class of a new row. This kind of task is called a classification task. We will solve it by using a classification algorithm.

But first, let’s take a step back and graph this data. We’ll create a scatterplot, with a horizontal axis representing the property size, and a vertical axis representing the property price.

We will draw datapoints representing entire homes in red, and ones representing private rooms in blue. If you forget, just mouse over any datapoint to see its class. Let's also include a few more datapoints from the same dataset, to make this more interesting.

Scatterplot!

Holy dog, a scatterplot! Now that we have the data plotted, let’s introduce the K-Nearest Neighbors algorithm to solve our classification task.

Mystery Datapoint

Let’s say we have this new datapoint— a property that sold for $XXX thousand, and has 720 square feet of space.

The K-Nearest Neighbors algorithm will look at the nearest neighbors of this new datapoint, and will tally up how many of them are houses, and how many are apartments.

K Neighbors

How many neighbors will KNN look at?

That’s what K stands for! It’s the number of nearest neighbors to tally up! Let’s say that K is 5 for now.

How does KNN know which datapoints are the nearest neighbors?

It calculates the distance A common way to calculate distance is using the euclidean distance formula you might remember from high school algebra. from the new datapoint, (at location $XXX k, 5) to every other datapoint on the grid. Then it chooses the top 5.

Majority Wins

Then it will take the majority vote of those 5 datapoints, and that is the algorithm’s guess about what the mysterious new datapoint is.

In this case, 4/5 of the nearest neighbors are apartments— so our best guess is that this new datapoint is an apartment.

What Happens if we Change K?

K is the model hyper-parameter that determines how many neighbors to take into account. We get to pick what K is. With two possible classes in our dataset, we can choose K to be any odd number, from 1 up to the size of the entire dataset.

Use the slider below to play with the value for K. Notice how it affects the model’s guess about the class of the mystery datapoint. This is called tuning the model parameters, and is a big part of creating accurate machine learning models.

What a Simple Model!

We'll dive deeper into model parameter tuning in future lessons. For now, let’s just assume that we chose to use K = 5 for our model.

When you first learn about KNN, it might seem wild that this could possibly work. It seems too basic, too reductive.

How well does the model actually work? And how do we measure that?

Testing our Model's Accuracy

Let’s remove the labels (house/apartment) from 100 of our 500 datapoints, and run the algorithm on each of those 100 “mystery” datapoints. We know what their true labels are supposed to be, and we have KNN’s guesses about what the labels should be. So, we can count how many it got right.

After doing this, we find that it guesses correctly on 75% of the datapoints! You would expect it to get 50% of them correct just by chance, so it’s way better than chance. This metric is called accuracy, and is one way to measure the performance of an algorithm.

Accuracy isn’t very sophisticated, but it will be good enough for now. We’ll dive into measuring algorithm performance more deeply in future lessons.
